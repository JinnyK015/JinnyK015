Problem-solving

Got blocked on a simple problem

I misunderstood what problem want to do. So, I tried to search other method. But it’s not work, and I read it again but still I didn’t understand this, so I ask my friends, and I realized I misunderstood the problems. I was a bit frustrate because my solutions were all different way. and it was a wasting time! So, I learn understand what problems is and what they want is the most important for solving problem.

Solved a problem in an elegant way

I’m not sure …have I solved problem in an elegant way? ...I don’t think so… But some problems were luckily, I searched MDN webpage some methods and there are exactly what I am looking for, so, I am using those methods and I figured it out! For that I learn… if I understand problems and if I know which method is good for those problems, it makes a smooth solution.

Reflect on how confident you feel using the problem solving techniques and process

I think pseudocode is so important for checking what am I understand, and I can plan for which method I need. It reduces my trying something then failed experience. Rubber ducky method is so great, I quite often using this too. And of course, I tried almost all of method using console.log when I did kata practice, I opened new repl.it and check console.log and read the errors. It is like a compass for me. Google is my best friends when I am coding. I have been focusing on trying to find answers myself, so I haven’t asked my peers or coaches for much help.
